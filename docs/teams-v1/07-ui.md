# 07 · UI reshape

Reference mock: `mocks/ui-v2/index.html`. This document is the authoritative spec; the mock is illustrative.

Reviewer checklist: UI work is reviewed against [`10-review-invariants.md`](./10-review-invariants.md), especially `UI-*` invariants.

## Screens

Four surfaces, no more:

```
Home              — list of chains
Needs attention   — approvals + blocked + merge decisions
Chain view        — chat + team roster + workspace (if VCS)
Settings          — templates, kinds (RO), providers, memory browser,
                    raw agents, direct-agent-chat (debug), daemon URL
```

Top nav has one badge, next to `Needs attention`, showing count of items awaiting the user.

## Sidebar

- Projects tree.
- Each project expands to its chains (direct children).
- Each chain shows status pills: `active | paused | planning | blocked | completed | needs attention`.
- `+ New chain` under each project.
- `+ New Project` at the bottom.
- **No** `+ Start team` button anywhere on the main path.

## Home

- List of chains grouped by project (or sorted by attention).
- Each row: title, status pills, project, coordinator, one-line meta (`5 / 39 done · 2 blocked · 2 review-ready`), `Open` button.
- Prominent `+ New chain` button at top.
- **No** cross-chain live-agent table (moved to Settings › Agents (raw registry)).
- **No** cross-chain approval panels (they live in `Needs attention`).

## Needs attention

Three sections in fixed order:

1. **Approvals** — approvals waiting on user, including merge decisions (with inline VCS box: branch, file list, `Show full diff`).
2. **Blocked** — blocked tasks/chains with the reason and one-click actions:
   - `Message coordinator`
   - `Open chain`
3. **Nudges due** — informational; agents the daemon just booted or is about to nudge.

Each item shows chain title in bold, subline with agent + reason, and inline action buttons.

## Chain view

Header:

- Breadcrumb → chain title + status pill.
- Key/value block: coordinator, progress, code location (if different from chain-host project), workspace path (if VCS).
- Actions: `Pause chain` / `Complete chain` / link to `Needs attention` filtered to this chain.

If VCS: **Workspace box** rendered between header and split panels.

Split into two panels:

### Left panel — coordinator chat

- Chat feed with the coordinator only.
- Every user message goes only to coordinator's inbox (chat routing rule A from planning).
- Smart-reply cards from coordinator inline (approve/reject/show diff).
- Coordinator can forward from another agent, rendered as `principal · forwarding coder`.
- Composer at bottom; hint copy: *"Sending a message boots the coordinator if it isn't running. Opening this chain view also warms the coordinator in the background. Live agents shut down after 30 min idle."*

### ChainTaskSurface

Rendered directly in Chain view before the chat/roster split. This replaces the removed global Pending Tasks tab.

- Shows tasks for the current chain only.
- Renders as a dependency-ordered todo list so the chain reads like an implementation/design plan.
- Active tasks are shown first; terminal tasks (`approved`, `done`, `completed`, `cancelled`, `archived`) are grayed out and moved to a separate **Completed tasks** section.
- Each collapsed task row shows only an order number, task title, perceived-status badge, and a right-side `Action needed` button when user action is required.
- Clicking a task expands inline `TaskDetail` scoped to the same chain.
- Local mutations show progress/error/success feedback and refresh Redux task/log state; no manual page reload is required.

### TaskDetail

Inline expansion opened from a ChainTaskSurface task row.

- Shows task status, assignee, reviewer, dependencies, description, votes/review history, and relevant metadata.
- Comments are behind their own expandable control so the plan remains easy to scan.
- Supports adding comments.
- Supports permitted status/actions such as start, done/review, block, later.
- Supports LGTM/NGTM voting where applicable.
- Supports task nudge using the task's chain-scoped assignee/reviewer/team-member route; it must not perform global role lookup.
- Every action must update local Redux state and/or immediately refetch task/log data so the result is visible without manual refresh.

### Right panel — team roster

- Compact list, one row per team member.
- Each row: agent name, current task (if any), state (`live · running Xm | live · reviewing | idle · will boot when …`).
- Blocked rows shown red with the blocking reason.
- Roles that haven't booted show `will boot when <trigger>` in grey.
- Clicking a row opens a read-only "what is this agent doing" side sheet (task + last 3 comments). No composer.

### Workspace box (VCS chains only)

Above the split panels. Fields per [`04-vcs.md`](./04-vcs.md) §UI surfacing:

- Branch/change name + `(base: main, N commits ahead)`.
- Path.
- Working tree summary.
- Changed files list with `M/A/D/?` flags + `+adds −dels`.
- `Show diff` toggle → inline collapsible diff panel with file picker.
- Buttons: `Refresh` / `Pull base` / `Preview merge`.
- Disclaimer: *"No VCS command runs without your click."*

## Modals

Only three:

- `+ New chain` — team-type-first creation. Required: Project (preselected when launched from a project row), Kind/team type, VCS on/off toggle/default. Optional: Title and Goal/description, available as "add details now" fields. Scaffold/template selection is not required at creation; the coordinator can apply task-bundle templates later after discovery.
- `+ New Project` — Name, Description, Repo path, VCS auto-detect.
- `Merge decision` — reachable from `Needs attention`; renders the workspace box + `Merge` / `Keep worktree` / `Abandon` buttons.

## Settings

Sidebar list, right pane is the selected item:

- Agent templates
- Team kinds (read-only)
- Providers & model tiers
- Memory browser
- Agents (raw registry) ← debug affordance
- Direct agent chat (debug) ← debug affordance
- Daemon connection

The debug affordances are labeled as such. Not on the main path.

- `Agents` top-level tab — moved to Settings.
- `Memory` top-level tab — moved to Settings.

- `Pending Tasks` tab — merged into per-chain board and `Needs attention`.
- `Templates` top-level surface — moved to Settings.

## Invariants

- **UI-1** No screen exposes "start a team" on the main path.
- **UI-2** No screen exposes a direct-to-agent chat composer on the main path.
- **UI-3** `Needs attention` is the single funnel for approvals, blocks, and merge decisions.
- **UI-4** Sidebar `+ New chain` action is available under every project.
- **UI-5** Chain view chat composer sends only to coordinator.
- **UI-6** Workspace box appears iff chain has a `vcs_workspace_id`.
- **UI-7** Every interactive element has a `data-debug-id` per AGENTS.md convention.
- **UI-8** UI data freshness uses Redux/HTTP/WS reconciliation; user-visible mutations must be visible without manual page refresh.
- **UI-9** Chain tasks are visible inside Chain view, and task detail exposes comments, votes/review history, assignee/reviewer/dependencies/status, status/review actions, and chain-scoped nudges.
- **WF-3** New chain UI requires only project/team kind plus optional VCS preference; title/goal are optional details.
- **WF-4** After submit, the new chain opens active/ready with one coordinator discovery task.
- **WF-5** Task-bundle templates are not mandatory in NewChainModal; they are coordinator helpers after creation.
