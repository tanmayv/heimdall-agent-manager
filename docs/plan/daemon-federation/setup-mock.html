What changes vs today

Both surfaces reuse existing components; no new modals.

  • Settings → Daemons pane gains a Peer daemons subsection (rows + add-card reuse the exact settings-daemon-row / add-daemon styling). Peers are linked, not switched to — distinct from the single active daemon.
  • Peer row meta shows connectivity + how many agents it lends; a disconnected peer is dimmed and its remote agents go offline.
  • AgentPicker gains a Remote · <peer> section and a remote row kind, reusing the row template. Only the peer agent's own instance id + daemon name are shown; the local proxy id stays hidden.
  • No "Start with" on remote rows — provider/tier belong to the peer.
  • Selecting a remote row behaves like the identity row: creates a dormant proxy behind the scenes and returns a concrete agent_instance_id to the role; callers are unchanged.
  • v1 scope: remote agents appear only in the reviewer/assignee picker.
